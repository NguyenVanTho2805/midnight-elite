## 2. Authentication System (JWT + bcrypt)

Approach: JWT thuần với `jose` + `bcryptjs` — không dùng NextAuth.  
Session: Cookie `tsix_session` (HttpOnly, SameSite: lax, 7 ngày)  
Route protection: `proxy.ts` (Next.js 16 — thay `middleware.ts` đã deprecated)

**Files:**
- `src/lib/session.ts` — encrypt/decrypt JWT, createSession/deleteSession/getSession
- `proxy.ts` (root) — bảo vệ `/student/*` và `/admin/*`
- `src/lib/auth-guard.ts` — `requireAdmin()`, `requireSuperAdmin()`, `isNextResponse()`

**API Endpoints:**

| Endpoint | Chức năng |
|---|---|
| `POST /api/auth/register` | Tạo tài khoản + gửi email xác thực |
| `POST /api/auth/login` | Đăng nhập, tạo session cookie |
| `POST /api/auth/logout` | Xoá cookie |
| `GET /api/auth/me` | Lấy user hiện tại từ cookie |
| `GET /api/auth/verify-email?token=` | Xác thực email, tự động login |
| `POST /api/auth/resend-verification` | Gửi lại email xác thực |
| `POST /api/auth/forgot-password` | Gửi link reset (hết hạn 1 giờ) |
| `POST /api/auth/reset-password` | Đổi mật khẩu + tự động login |
| `POST /api/auth/change-password` | Đổi mật khẩu (cần mật khẩu cũ) |

## 7. Payment Webhook (SePay + VietQR)

```
POST /api/payment/create → PendingOrder + mã TSIX-XXXXXX + QR VietQR URL
→ Học viên quét QR chuyển khoản ngân hàng
→ SePay webhook → POST /api/webhook/payment
→ Match mã TSIX → tạo Order + Enrollment → email xác nhận
→ Xoá PendingOrder
```

Env cần thêm: `SEPAY_WEBHOOK_TOKEN`, `BANK_ID`, `BANK_ACCOUNT`, `BANK_NAME`
