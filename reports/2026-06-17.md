### 2.5 prisma.ts — Lazy initialization fix
**Vấn đề:** `createClient()` được gọi trực tiếp khi module load:
```ts
export const prisma = globalForPrisma.prisma ?? createClient(); // gọi ngay!
```
Khi Next.js build (Turbopack "Collecting page data"), module được evaluate trước khi DATABASE_URL inject → `new URL(undefined)` → `TypeError: Invalid URL`.

**Fix:** Proxy-based lazy singleton — `getClient()` chỉ chạy khi có request thật:
```ts
export const prisma = new Proxy({} as PrismaClient, {
  get(_, prop) {
    return (getClient() as any)[prop]; // getClient() gọi lần đầu khi prisma.* được dùng
  },
});
```
