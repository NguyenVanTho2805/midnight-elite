# Báo cáo phiên làm việc — 12/06/2026

## 1. Fix đăng ký "Lỗi server" 500

**Root cause:** Turbopack cache `.next/` chứa bản compile cũ của Prisma client — bản cũ không có `parentPhone` (và các field mới thêm vào schema). Mọi `user.create()` call đều bị Prisma runtime validation reject với lỗi "Unknown argument `parentPhone`".

**Fixes áp dụng:**

1. Xóa `$queryRaw SELECT nextval(...)` khỏi register route — thay bằng PostgreSQL column DEFAULT (`nextval('student_id_seq')`) đã được setup script gán sẵn trong DB. Prisma omit field → DB tự gán `studentId`.
2. Xóa `.next/` cache + kill node process + restart dev server để Turbopack recompile với Prisma client mới.
3. Schema update thử dùng `@default(dbgenerated("nextval('student_id_seq')"))` nhưng `db push` fail vì sequence đã có dependency → giữ schema `Int? @unique` không có `@default`, DB vẫn dùng DEFAULT đúng.

**File thay đổi:** `src/app/api/auth/register/route.ts`

**Lưu ý cho sau:** Khi thêm field mới vào Prisma schema và chạy `prisma generate`, phải xóa `.next/` và restart dev server — Turbopack không tự recompile Prisma client.

## 2. Module Tra cứu học viên (`/tra-cuu`)

### API — `src/app/api/tra-cuu/route.ts`
- POST `{ phone }` — validate format SĐT Việt Nam (`0[3|5|7|8|9]XXXXXXXX`)
- Tìm đồng thời bằng `OR: [{ phone }, { parentPhone: phone }]` — không cần chọn loại SĐT
- Trả về: `name`, `studentId`, `courses[]` (id, name, shortTitle, category, status)
- Rate limit 10 req/phút/IP
- Không trả về: email, password, phone đầy đủ

- Kết quả: avatar letter + tên + badge `HS-XXXXXX` + danh sách khóa học với tag màu theo category
- States đầy đủ: loading / not found / error / success
- Notion flat design, nhất quán với toàn bộ guest pages
