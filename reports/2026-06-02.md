## 2. Phân biệt Học viên vs Chưa kích hoạt — Áp dụng toàn hệ thống

Triết lý:
- **Học viên** = `role: "student"` + có ít nhất 1 enrollment
- **Chưa kích hoạt** = `role: "student"` + không có enrollment nào

### Dashboard
- Health card "Tổng học sinh" có dòng sub: `"X học viên · Y chưa kích hoạt"`
- Content admin thấy: "Học viên đã KH" + "Cần chú ý (GPA < 7)" thay vì "—"

### Hồ sơ học sinh
- Subtitle: `"X học viên · Y chưa KH"` thay vì `"N học sinh"`
- Thêm state `roleFilter` + cập nhật `filtered` + `counts`
- Thêm row filter **Tất cả | Học viên (X) | Chưa KH (Y)** trên row status filter

### Quản trị viên
- Stats bar: `"Học viên: X | Chưa KH: Y"` (trước đó là `"Học viên: N"` gộp hết)
- Filter tabs: thêm tab **Chưa KH** (lọc `enrollments === 0`)
- `RoleBadge`: dùng `_count.enrollments` để phân biệt badge Học viên (xanh) vs Chưa KH (xám)

### Doanh thu
- Card label: `"Học sinh"` → `"Học viên đã KH"` — rõ nghĩa hơn

## 3. Admin `/quan-tri-vien` — Audit 4 vòng

### Vòng 1 — Code & logic
- Merge 2 import cùng `@/contexts/AuthContext`
- `filtered` + counts không memoized → thêm `useMemo`
- 3 `.filter()` riêng → 1 `useMemo` single-pass (`counts`)
- Thiếu `credentials: "same-origin"` trên fetch
- `RoleBadge` hiện "Học viên" cho cả chưa kích hoạt → fix dùng `enrollments`
- `changeRole` spread `...data` mất `_count` → chỉ update `role/adminRole`
- Xóa `createdAt` dead field khỏi `UserRow` interface

### Vòng 2 — API consistency
- `GET /api/users`: raw session check → `requirePermission(MANAGE_ADMINS)`
- `PUT /api/users/[id]/role`: raw session check → `requirePermission(MANAGE_ADMINS)`
- Xóa `createdAt` khỏi API select (page không dùng)
- Xóa `name, email` khỏi PUT response select (page không dùng)
- `changeRole`: thêm `try/catch/finally` — `setPending(null)` luôn chạy dù lỗi mạng

### Vòng 3 — API sâu hơn
- `PUT` gọi `getSession()` 2 lần: `requirePermission` đã trả về session → dùng `auth` trực tiếp
- `PUT` không validate `role` input → thêm `ALLOWED_ROLES` + `ALLOWED_ADMIN_ROLES` allowlist
- `findUnique` thừa → xóa, dùng `try/catch` trên `prisma.update()`
- `fetch("/api/users")` không check `r.ok` → crash khi lỗi → thêm guard `Array.isArray`

### Vòng 4 — Error handling
- `catch` block maps tất cả DB errors → 404 → sai với connection error / timeout
- Fix: `Prisma.PrismaClientKnownRequestError` + `e.code === "P2025"` → 404; còn lại → 500

### UX polish cuối
- Stats bar: skeleton loading thay vì "0 Tổng" trong lúc chờ
- Cột "Khoá học": admin hiện `"—"` (xám), student hiện số khoá (xanh)
- Filter buttons kèm count: `"Học viên (8)"`, `"Chưa KH (2)"` — nhất quán với hoc-sinh
